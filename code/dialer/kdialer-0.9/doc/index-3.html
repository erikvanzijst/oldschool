3. Fundamentals

KDialer is very simply to use. If you have ever used Micro$oft's dialer.exe, you will have no problems with KDialer.

3.1 Three ways to dial a number

In order to dial a phonenumber, you can simply type it in the textbox and press enter or click dial. You can also click the buttons on the keypad or just choose one of the programmable buttons in the "Speed dial" group.

3.2 Store phonenumbers

To program the eight "Speed dial" buttons, click "Edit speed dial" in the edit menu. Now the edit window pops up. By default, the first button is selected for editing, but ofcourse any button can be edited from here by clicking it.
As you type the name, it will appear on the selected button. Then type the phonenumber and click save or press enter. Hit "Esc" to return without saving.

3.3 Using a dial-prefix

In some places you first need to dial a special prefix digit in order to dial outside numbers. For example, an "0" (zero) is very common and often used as prefix.
To always use a prefix, go to the Options menu (in the edit menu, or press Ctrl-O) and enter the prefix digit(s) in the textbox.
After pressing enter or clicking Save, KDialer will always prepend your prefix to every number you dial.

3.4 Specifying the modem device

By default, KDialer will use "/dev/modem" to access your modem. If your system does not have this link properly set up, or if you want to explicitly use another device, go to the Options menu (Ctrl-O) and enter your own device. (For example: /dev/cua1 for "COM2" on Linux 2.0.XX kernels, or /dev/ttyS1 on 2.1.XXX kernels.)
